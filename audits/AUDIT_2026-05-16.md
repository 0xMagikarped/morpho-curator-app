## 0. Repo map

**Not Next.js.** This is a **Vite 7 + React 19 + TypeScript SPA**, deployed on Vercel as static
files (`vercel.json` rewrites everything to `index.html`). There is **no `app/`/`pages/` router,
no middleware, no Server Actions, no Edge runtime, and no `api/` directory**. Consequently a large
fraction of "Pass C" Next.js checks are structurally N/A (called out explicitly, not invented).

- **Size:** 223 TS/TSX files, ~41.4k LOC (`src/`).
- **Server/client boundary:** none — 100% client-side. All RPC and wallet interaction happens in
  the browser. `@vercel/node` is a dependency but unused (no functions).
- **Web3 stack:** wagmi `^2.19.5`, viem `^2.47.0`, `@rainbow-me/rainbowkit` `^2.2.10`, Morpho SDKs
  (`blue-sdk` 5.17, `blue-sdk-viem` 4.5, `simulation-sdk` 3.4, `liquidity-sdk-viem` 1.4).
  **No web3.js / web3-utils / ethers** (clean — CVE-2024-21505 N/A).
- **Transports:** `src/config/wagmi.ts` — `fallback([...public RPCs])` for SEI(1329), Ethereum(1),
  Base(8453), BSC(56), Pharos(1672). No paid keys, no server proxy. Optional `VITE_*_RPC_URL`
  env overrides.
- **Morpho contracts:** single canonical `src/config/chains.ts` (`CHAIN_CONFIGS` by chainId) —
  Morpho Blue, V1/V2 factories, public allocator, V2 adapters, Moolah/Lista. Supports MetaMorpho
  V1, Vault V2 (adapters), and Moolah vaults.
- **Data layer:** Morpho GraphQL (`src/lib/data/morphoApi.ts`) with RPC fallback
  (`src/lib/data/rpcClient.ts`, 1025 LOC god module); TanStack Query v5 + Zustand v5; query-key
  factory in `src/lib/queryKeys.ts`.
- **Env surface (all `import.meta.env.VITE_*`, build-time inlined into client JS):**
  `VITE_WALLETCONNECT_PROJECT_ID`, `VITE_SENTRY_DSN`, `VITE_{ETH,BASE,SEI,BNB,PHAROS}_RPC_URL`,
  `VITE_BNB_MARKET_FACTORY` (used at `chains.ts:236`, **missing from `.env.example`**). Build-only
  `process.env.SENTRY_*` in `vite.config.ts` (not bundled).
- **Tooling state:** `tsc -b` clean (0 errors). 70 unit tests pass (4 files). jscpd 2.31%
  duplication. CI (`.github/workflows/test.yml`) runs **only `npm run test:run`**.

## 1. Executive summary

- **Total findings: 41** — BLOCKER: 0, HIGH: 10, MEDIUM: 18, LOW: 8, INFO: 5 (positive controls).
- **No true BLOCKER triggered:** the prompt's BLOCKER conditions (unpatched Next CVE, paid RPC key
  in bundle, private key in repo) were each checked and are **clean/N/A**. However, the four HIGH
  clusters below should be treated as **release-blocking for a production financial tool**.
- **Top 3 risks right now:**
  1. **On-chain write safety (HIGH).** Most vault-governance writes (caps, queues, fees, timelock,
     roles, V2 adapters, registry) are broadcast with **no `simulateContract` preflight**, and
     **not one ABI in the repo contains a single `type:'error'` entry** — so every revert surfaces
     as opaque "execution reverted". Curators sign powerful txs blind and learn nothing when they
     fail. (D4, D5; also D2 chain-guard gaps.)
  2. **Frontend/wallet security posture (HIGH).** CSP is `Content-Security-Policy-Report-Only`
     (never enforced) with `script-src 'unsafe-eval'`; no HSTS; no Permissions-Policy/COOP. This
     is precisely the attack class (XSS/exfil/MITM/DNS-hijack) that owned Curve/Aerodrome/Cream.
  3. **Silent data trust (HIGH).** Zero runtime validation (`zod` is not even a dependency) of
     Morpho GraphQL / DefiLlama responses feeding allocation & health math; multiple RPC `catch`
     blocks conflate "RPC failed" with a *safe* answer — `checkMarketExists` returns "market does
     not exist" on RPC error, gating market creation. Plus 10 HIGH npm advisories + all-`^` ranges
     with lifecycle scripts enabled (Shai-Hulud class).
- **Confidence:** High for A/B/D (direct code reading + tool corpus). Medium for C — bundle/secret
  analysis is verified against a *current* `dist/` build (post-dates source), but Vercel dashboard
  settings (Deployment Protection, env scoping, domain/DNS hardening) **cannot** be verified from
  the repo and are listed in §8.

> **Reconciliation note:** the Sentry-PII-leak path (vault addresses in breadcrumbs) lives in
> `src/lib/monitoring/txMonitoring.ts`, which knip confirms is an **orphan (imported nowhere)**.
> It is therefore a *latent dead-code* hazard, not an active production leak — graded MEDIUM, not
> HIGH, accordingly. `src/lib/sentry.ts` itself *is* wired (via `main.tsx`) and its `beforeSend`
> sanitizes only `event.message`, which remains a real (if currently low-exposure) gap.

## 7. Architectural bets (this quarter)

1. **A typed write pipeline with mandatory simulation + decoded errors.** One `useVaultWrite` path: build intent → `simulateContract` → decode custom error → state-diff preview → execute. Make unsimulated writes impossible by construction.
2. **A validated data layer.** `zod` schemas for every external source (Morpho API, DefiLlama, RPC decode), a "could not verify" first-class state, and the rule that an RPC failure never resolves to a safe-looking value.
3. **Split the `rpcClient.ts` god module** into a thin client + per-domain read modules, and consolidate the two hook roots — recover testability on the financial core.
4. **Decision-grade reads.** `fallback({ rank:true })`, `blockTag:'safe'` for cap/oracle decisions, optional paid-RPC env path, and a thin server read-proxy so risk decisions don't trust a single public RPC.
5. **Frontend-integrity program.** Enforced CSP without `unsafe-eval`, SRI, HSTS preload, registrar lock + DNSSEC, an IPFS/ENS mirror, and DNS/Sentry monitoring — directly countering the dominant 2024-26 DeFi frontend exploit class.
